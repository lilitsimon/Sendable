Privacy Policy for Email and Message Proofreader: Sendable

Sendable helps users proofread and improve emails, messages, and other professional writing without changing their voice.

Effective date: May 24, 2026

What Sendable collects

Sendable may collect and process the following information when a user chooses to use the product:

  • Text that a user pastes into the extension
  • Text that a user selects on a webpage and explicitly chooses to pull into the extension
  • An email address submitted during onboarding
  • Basic technical request data such as timestamps, origin, and user agent when requests are sent to Sendable’s backend

User-submitted text may include personal communications, message drafts, emails, professional notes, or website content that the user intentionally chooses to process.

How Sendable uses data

Sendable uses collected data only to:

  • Provide proofreading, grammar correction, spelling correction, clarity improvements, and tone refinement
  • Return an improved version of the user’s text
  • Support onboarding and track extension signups
  • Maintain, secure, troubleshoot, and improve Sendable’s core functionality

Sendable does not use user-submitted text or onboarding email addresses for advertising, credit scoring, or unrelated profiling.

How text is processed

When a user asks Sendable to improve text, that text is transmitted securely to Sendable’s backend and may be sent to an AI processing provider strictly for the purpose of generating the improved version.

Sendable processes user-submitted text only to provide proofreading and message improvement.

How email addresses are handled

If a user submits an email address during onboarding, Sendable may send that email address to Sendable’s backend to track extension users and support product updates, support, or communications where applicable.

Depending on Sendable’s configuration, onboarding email submissions may be stored in backend logs or forwarded to a secure storage or webhook destination operated by Sendable.

What Sendable does not collect

  • Sendable does not collect browsing history.
  • Sendable does not monitor pages in the background for unrelated purposes.
  • Sendable does not read webpage text unless the user actively pastes it, selects it, or explicitly invokes Sendable’s features.
  • Sendable does not sell user data.

Data sharing

Sendable may share data only with service providers necessary to operate the product, such as:

  • Hosting and backend infrastructure providers
  • AI processing providers used to generate proofreading results
  • Optional storage, analytics, or webhook providers used by Sendable to store onboarding submissions or operate the service

Sendable does not share user data with data brokers or advertisers.

Data retention

User-submitted text is processed only as needed to provide the requested output. Onboarding email addresses and related submission records may be retained for product operations, analytics, support, or updates unless deletion is requested and applicable.

Security

Sendable uses modern encryption and secure transmission methods when sending data between the extension, backend services, and processing providers.

User choices

  • Users choose what text to submit for improvement.
  • Users may choose whether to submit an onboarding email address.
  • Users can stop using the extension at any time.
  • Users may request deletion of stored onboarding or related user data, if applicable.

Contact

For privacy questions or data deletion requests, please use the support contact listed in the Chrome Web Store listing for Sendable or your published product support channel.

Chrome Web Store disclosure

Sendable’s use of information received from Chrome and user-submitted content is intended to comply with the Chrome Web Store User Data Policy, including the Limited Use requirements.

This privacy policy should be kept aligned with Sendable’s real backend behavior and storage setup. If Sendable begins storing onboarding email addresses in a database or forwarding them to a specific provider, this page should be updated to reflect that exact behavior.
